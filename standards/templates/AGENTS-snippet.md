## Standards Read Order

Before planning or implementing work, read documents in this order when they exist:

1. `AGENTS.md`
2. `docs/project-memory.md`
3. relevant files under `docs/adr/`
4. project architecture and design documents

## Standards Package

This repository follows a reusable engineering standards package covering:

- comment and Javadoc rules
- ADR format
- project memory format
- review expectations for durable decisions

When changing a durable policy or architectural default, update `docs/project-memory.md` and the
relevant ADR.

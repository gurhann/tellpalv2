# PostgreSQL 15 Flyway Checklist

## Naming

- migration file: `V{number}__short_description.sql`
- tables and columns: snake_case
- constraints: `pk_<table>`, `fk_<from>__<to>`, `uk_<table>__<column>`, `chk_<table>__<rule>`
- indexes: `idx_<table>__<column_list>`

## Table Defaults

- prefer `bigint generated by default as identity primary key`
- prefer `timestamptz` over `timestamp` for server-side event and audit times
- use `text` unless a real length limit matters to the domain
- default required columns to `not null`

## Constraints

- primary key on every table
- foreign keys for aggregate references and lookup tables
- unique constraints for business keys and stable external identifiers
- check constraints for bounded status sets, non-negative values, and valid date/range rules

## Indexing

- index foreign-key columns
- index frequently filtered status, slug, external_id, and timestamp columns
- avoid duplicate indexes created implicitly by primary key or unique constraints
- use composite indexes only when query patterns require left-to-right matching

## Migration Discipline

- write forward-only SQL for Flyway
- keep one migration focused on one schema change set
- avoid mixing data cleanup with broad DDL unless the migration requires it
- call out locking or backfill risk when altering large tables
